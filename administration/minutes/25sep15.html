Topics discussed during meeting:

Customer Questions

  • What programming languages does the test taking IDE need to support?

  1. Handled by configuration of room. The tool does not need to implement machine restrictions.

  • Do the test-takers need an “account”? Should they be able to view past tests they took or retake a test? What other sorts of information should they be able to view?

  1. Exists under proctored exam configuration

  • How editable should the generated tests be, and in what ways?

  1. Total freedom. Make the program a total sandbox.

  • What should the user be able to search for a test by? (name, date, author, etc…)

  1. Only for practice tests.

  • Can multiple people take the test at the same time?

  1. Yes. This should be a supported feature.

  • How does teacher get student’s results?

  1. The teacher should be able to view student scores after the proctored session is complete.

  • When can we show students the answers?

  1. Couple of options: See your auto-graded questions immediately after you take it, knowing that this is approximate. Alternatively students can not see any grades.

  • Searching for questions?

  1. Questions have attributes, search based on those primarily. Alternatively use a filtered text search for questions?
