1.2. System Personnel

The personnel involved in the Test Tool project are organized into the following groups and subgroups:

  1. end users

    1. students

    2. teachers

    3. system administrators

    4. unregistered individual users

  2. primary customer

  3. developers

End users are those who use the Test Tool for test creation or test taking purposes. Students are users that will use the tool to take published exams. Teachers are users that will create exams using the tool, they will have an option to contribute questions to the test bank. System administrators are users who will deploy the test tool and setup the testing environment. Unregistered individual users are users that would like to view publicly published exams.

The primary customer is Gene Fisher. He is the customer representative for his faculty and staff colleagues in the Computer Science department at the California Polytechnic State University, San Luis Obispo. Fisher will gather any product requirements and suggestions from end users. He will need to approve any requirements before development begins.

The primary system developers are: Zach Arend, Jameson Li, Andy Dufrene, Thomas Steinke, David Ellison, and Greg Sawers. Development activities are all those of the software development process, from requirements analysis through product implementation and deployment. The developers will also conduct the ongoing process activities of testing, configuration, documentation, and project management.
The Test Tool is intended to serve as a prototype developed in a software engineering course. The goal is for the above developers to outline the software development process needed to create the product, and also create a product prototype.
