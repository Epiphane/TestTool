2.2

After the instructor logs in, he or she selects the "Tests" button.

Figure 2: Tests Overview Page.

The tests view is empty because the instructor has just created an account, and has not created tests yet.

TestTool presents the instructor with test options available to him or her. On the top menu bar, the "Create Test" button will take the instructor to the test creation menu. The top menu bar has a search field, allowing the instructor to search for any tests they have already created by name.

The Edit button allows the instructor to edit an already created test. Discussion of this part of the application can be found in section 2.4. The instructor will use the edit button to change the most recent versions of the test, allowing them to change or add questions and save new versions.

The Publish button allows the instructor to publish the test for taking and viewing by students. A dialog (to be created) will allow the instructor to decide how this test should be published including the options to let students, view all questions, only view questions after taking the test, or only allow them to view that the test is published and ready to be taken. The general permissions of the test is indicated by the Public? shader, where a red box indicates that the test is not public, a green box indicating the test is fully public, and a yellow box indicating the test has limited permissions.

The Grade button allows the instructor to grade an already completed test. After the test has been taken by all students, the instructor is able to view the automated results from the grader, as well as view all answers made by students to each question. The grading screen will be discussed further in section 2.6.

The View button will take the instructor to a screen where they can view a test and information about that test. This screen will closely resemble how a student will see a test while taking it, but will include additional information such as question count, last time given, and number of versions amongst others. This screen will be discussed further in section 2.3
